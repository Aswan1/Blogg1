Vad är molnet?

Molenet är en samling IT tjänster som tillhanshålla över internet. och vad byteder detta? Jo det betyder att vi behöver inte köpa fysiska server längre


Vilka för delar och nackdelar kan molnet introducera?

Fördelar

  • En fördel med molnet är att vi slipper en stor del av ansvaret av den fysiska hårdvåra och alla stora molntjänst levrantör har bra garinti om säkerhet och downtime

  • Inga startkostnader man betaler bara för resurser man använder

  • Man behöver inte köpa och hantera dyr infrastruktur som kanske inte ens används fullt ut.


Nakdelar

  • Säkerhet
  • Tillgänglighet
  • Kan bli dyrare om man änvander det på fel sätt

Vad är en CI pipline

CI pipeline automatiserar mjukvara utvecklings process och bygger koden plus att kan köra automatiska testen kontinuerligt. CI pipline hjälper att vi kommer till produktion snabare


Hur har du implementet en CI pipeline i GitHub actions på ett eksisterende projekt, vilka steg har du gått igenom?

  • skapade en work flow
  • la till YAML file
  • definerade vilken steps som ska köras i YAML file
  • sedan pushade koden/vps/

Beskrivning din GitHub action workflow YAML fil.


YAML filen består av följande delar

  • name: namnet på workflow som ska visas /li>
  • on: när triggas jobb work flow
  • jobs: grupper alla samlar alla jobs
  • build:
  • runs-on: vilken miljö ska man köra, Windows eller Linux
  • steps: alla steps som kollar dåliga version som har fel
  • uses: actions/checkout@v2 den tar senaste version av din kon och läger den i runner
  • - uses: actions/setup-node@v1 laddar ner senaste versin av dotnet
  • run:

Vad har in installerat lokalt / Github

Docker

Hur har ni fått applikationen att kör i en container

Jag har fick den och köra i en container men det gick inte och köra den via consolen. Det gick inte heller och starta den via docker desktop.

Beskriv din docker fil

  • FROM mcr.microsoft.com/dotnet/sdk:3.1 AS build, här är imagen som den kommer att utgå ifrån /li>
  • WORKDIR /app här skapar katalogen app i roten av filsystemet inuti imagen
  • COPY publish . här kopierar innehållet i katalogen publish där vår webbapplikation ligger till nuvarande katalog i imagen (d.v.s. /app)
  • ENTRYPOINT ["dotnet", "SimpleWebHalloWorld.dll"] definierar vilket kommando som ska köras som standard när en container körs från den här imagen
